História dos computadores

(Voltar)

Dentro dessa aula introdutória superinteressante sobre a história dos computadores. Aprendemos que tudo começou bem antes da internet e dos PCs que a gente conhece hoje.Vimos que os primeiros dispositivos de cálculo foram bem simples, como o ábaco (usado a milhares de anos).

Depois, estudamos a evolução para máquinas mais complexas, como a Máquina Analítica de Charles Babbage no século XIX, que é considerado o "avô do computador moderno. Também falamos sobre Alan Turing, que criou conceitos fundamentais para a computação, principalmente durante a Segunda Guerra Mundial.

Na década de 1940, surgiram os primeiros computadores eletrônicos, como o ENIAC. Eles eram enormes e usavam válvulas. Com o tempo, vieram os transitores, os circuitos integrados e depois os microprocessadores, que deixaram os computadores menores, mais rápidos e mais acessíveis.

A gente também discutiu como os computadores evoluíram dos mainframes até os desktops e notebooks, e como hoje temos smartphones com mais poder de processamento do que muitos computadores antigos.

O que são Clients?

(Voltar)

Em termos simples, o client é o lado do usuário, ou seja, tudo que é executado no dispositivo de quem está acessando um site, como navegador (browser), interface gráfica, e ações como clicar e preencher formulários.

O conteúdo que aparece na tela, como HTML, CSS e JavaScript, normalmente roda no client. Isso significa que a experiência visual e interativa de um site depende bastante dessa parte. Também vimos que o client envia requisições para o server quando precisa de dados.

O que são Servers?

(Voltar)

O server (ou servidor) é o lado responsável por responder às requisições feitas pelo client. Ele armazena os dados, arquivos e códigos que fazem os sites funcionarem. Quando acessamos uma página, o client faz um pedido, e o server devolve a resposta, como um arquivo HTML ou informações do banco de dados.

O server pode ser configurado com linguagens como PHP, Node.js, ou Python, e é onde rodam os sistemas que tratam os dados do usuário, fazem cálculos e gerenciam conteúdos.

Como é uma estrutura básica de um HTML?

(Voltar)

A estrutura básica de um HTML é como o esqueleto de qualquer página web. Tudo começa com a declaração !DOCTYPE html, que serve para o navegador entender que estamos usando HTML.

Depois disso, usamos a tag "html" para abrir o documento. Dentro dela, temos duas partes principais: o head e o body.

  • No head, ficam as informações que não aparecem diretamente na tela, como o título da página, os metadados e os links para arquivos externos, como o CSS
  • No body, colocamos todo o conteúdo visível do site: textos, imagens, botões, links, etc.

Entendendo a diferença entre listas ordenadas e não ordenadas

(Voltar)

As listas ordenadas são aquelas que seguem uma ordem numérica ou com letras. Usamos a tag ol (de ordered list) e cada item vai dentro de uma tag li. Isso é útil quando a ordem dos itens importa, como em receitas ou passo a passo.

Já as listas não ordenadas são usadas quando a ordem não importa. Usamos a tag ul (de unordered list) e os itens também vão dentro da tag li. A lista aparece com marcadores (bolinhas).
